请尽量输出成可执行的条件表达，优先使用这些形式或同类形式：
- `always=true`
- `字段=true`
- `字段=false`
- `字段=某值`
- `字段!=某值`
- `字段>=阈值`
- `字段<=阈值`
- `字段A=某值 and 字段B=true`

只有在当前任务没有标准字段或证据暂时不足时，才允许输出中文说明型因素；这类因素应尽量接近用户可回答的问题，而不是空泛分类。

`evidence` 请尽量体现判断依据，优先说明：
1. 该工序在什么样本组合里稳定出现
2. 它在哪些样本中缺失、被替代，或仅在特殊边界下出现
3. 它在路线中承担什么作用、处于什么阶段、前后常与哪些工序联动
4. 为什么当前结论不是 `always=true`
5. 你判断该工序触发边界的主要依据是什么
6. 如果当前只能给 `WEAK`，当前最主要的不确定点在哪里
7. 如果没有足够证据支持任何条件，请明确说明“当前样本不足以锁定影响因素”

输出 JSON 数组：
```json
[
  {
    "name": "镗孔",
    "sequence": 120,
    "confidence": "STRONG",
    "factors": [
      {
        "name": "hole_complex=true",
        "evidence": "当前样本中，镗孔只出现在孔系较复杂、钻孔后仍需进一步修正孔形或尺寸的零件中；在仅有简单孔加工的样本中未出现，因此不能判为 always=true。",
        "strength": "STRONG"
      }
    ]
  },
  {
    "name": "去应力",
    "sequence": 310,
    "confidence": "WEAK",
    "factors": [
      {
        "name": "热后尺寸稳定或变形控制要求=true",
        "evidence": "该工序仅出现在部分热后样本中，且多与后续精整或最终质量控制联动；当前样本提示它可能与热后尺寸稳定有关，但边界仍不足以锁定为强规则，因此暂不宜输出 always=true。",
        "strength": "WEAK"
      }
    ]
  }
]
```

补充示例口径：
- 对用户可追问的 `WEAK` 因素，优先写成“后续可确认的边界”，例如：
  - `IT7 / IT6 / IT5 及更高精度等级`
  - `按材料牌号区分`
  - `按材料大类区分`
  - `按热处理技术条件区分`
- 不要优先写成过于系统化的大类总结，例如：
  - `由加工要求差异导致`
  - `由材质差异导致`
  - `需要通过磨削保证`
- `evidence` 文案尽量像工艺人员在解释原因，而不是系统提示语；应更接近：
  - `当前样本里，磨外圆主要集中在 IT7 及以上、且热后还要恢复尺寸的外圆段`
  - `当前样本里，正常化主要集中在这几类材料上，和调质的材料范围并不相同`
