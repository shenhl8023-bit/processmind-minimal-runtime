问题口吻要像审核人员在确认边界，不像聊天，也不像系统播报。

统一框架：
- 先问这道工序存在的条件依赖于什么
- 再问具体边界

写法要求：
1. 一次只问一个边界，不混多个维度。
2. 先从工序名称和工序内容中判断“加工对象”和“加工方法”。
3. 已知加工对象时，不再重复确认对象是否存在。
4. 特征成形类工序优先问特征、角色或边界；要求驱动类工序优先问要求边界。
5. 第二问进入具体边界后，优先保留精度等级、粗糙度等级、材料牌号、材料大类、热处理类别等可直接选择的具体项。
6. 尺寸边界尽量问成最大直径、长度、壁厚、体积或热处理深度这类具体口径。
7. 单个工序不要写成组合名工序的问法；组合名工序先问统一名称，再问存在条件。
