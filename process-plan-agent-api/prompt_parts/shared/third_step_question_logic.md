第三步问答逻辑：

1. 全覆盖工序不提问；非全覆盖工序必须提问。
2. 组合名工序先确认统一名称；统一名称后若已全覆盖，则直接结束；否则再问存在条件。
3. 第二问及后续追问，只能沿第一问的已选方向展开。
4. 不能停留在“要不要”或“是否存在”的口头判断上，必须继续追到可判断的具体条件。
5. 只有当回答已经足以写成规则句时，才可以结束。

不同工序类型的第一问入口：
- 粗加工、半精加工：结构或加工对象
- 精加工、超精加工、光整加工：部位 + 精度类型
- 热处理、预处理：材料或工艺条件
- 检验类：阶段转换或质量控制要求
- 清洗、去毛刺：后续工序或转序需求
- 倒角：边缘处理或导入需求
- 标印：标识或追溯需求
- 包装：交付、防护或转运需求

补充边界：
- 单个工序按单个工序发问，不能误用组合名工序的话术。
- 普通孔加工工序默认先按孔类结构或孔加工对象发问，不默认归入精加工精度树。
- 工序依赖条件允许写成“当工序 A 存在时，这个工序保留”或“当工序 A 不存在时，这个工序保留”。
- 检验类工序优先按阶段转换或质量控制要求发问。
- 清洗、去毛刺等辅助工序优先按后续工序或转序依赖发问。

结束标准：
- 已确定该工序主要依赖哪一类条件
- 已细化到可判断的具体条件
- 该条件能够区分这道工序什么时候存在、什么时候不存在
- 已能够直接写成规则句
