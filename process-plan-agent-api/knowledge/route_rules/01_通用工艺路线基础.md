# 通用层：通用工艺路线基础

## 1. 适用范围

本层知识尽量保持跨零件族通用，用于帮助系统先建立规则导向的工艺理解框架。

它适合用来解释：

- 工艺路线为什么按阶段组织
- 哪些因素更可能导致路线变化
- 哪些工序更像主线，哪些更像按需补充

## 2. 通用工艺阶段模型

系统应优先把路线理解为按阶段组织，而不是单纯一串工序名：

1. 毛坯准备或预处理阶段
2. 粗加工阶段
3. 热处理或表面处理前后过渡阶段
4. 半精加工阶段
5. 精加工或定尺寸阶段
6. 检验、清洗、无损检查、表面处理收尾阶段

同名工序如果出现在不同阶段，通常应被视为不同阶段工序，而不是简单重复。

## 3. 通用规则因素框架

系统应优先从以下几个大类理解工艺路线变化：

- 材料因素
- 热处理或表面处理因素
- 结构因素
- 精度或配套要求
- 检验要求
- 工艺可行性和风险因素

这些因素的价值，在于它们能够解释：

- 哪些工序稳定出现
- 哪些工序只在部分样本中出现
- 哪些工序是按需补充工序
- 哪些工序会带出伴随检验或清洗链

## 4. 主线与按需工序理解

通常更接近主线的工序：

- 毛坯准备
- 预处理
- 粗加工
- 半精加工
- 精加工
- 阶段检验

通常更可能由条件触发的工序：

- 去应力
- 特殊热处理
- 特殊表面处理
- 专门的无损检查
- 特殊结构加工
- 后续补充精整工序

系统不应把所有低频工序都直接当噪声，也不应把所有低频工序都硬判为稳定主线。

## 5. 使用边界

本层知识适合：

- 解释阶段组织方式
- 解释规则因素大类
- 约束第二步的理解方向

本层知识不应用来：

- 替代当前项目证据
- 臆造当前项目不存在的工序
- 决定某个零件族的专有路线细节
