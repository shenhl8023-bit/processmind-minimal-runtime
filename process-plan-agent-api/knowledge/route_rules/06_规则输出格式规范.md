# 规则输出格式规范

> 说明：本文件用于约束第二步审核结果的目标输出格式；除非后端显式接入，否则不默认作为模型运行时主上下文。

## 1. 目标

本规范定义第二步对话结束后，每道工序应输出的结构化规则表达。

输出的目标不是描述工序本身，而是表达该工序出现或不出现的触发条件，使其能被规则库直接收录和使用。

## 2. 单道工序规则输出模板

每道被问答完毕的工序，应按以下模板输出一条规则：

```text
工序名称：{工序名}
触发因素类型：{材料类 / 热处理类 / 结构类 / 精度类 / 可加工性类}
具体触发因素：{从当前文档提取的实际值，或中类词}
规则表达：若 {触发条件}，则 {出现 / 不出现} {工序名} 工序
确认状态：{已判稳 / 待确认边界}
备注：{可选，补充说明边界或例外情况}
```

### 2.1 示例：精度类触发

```text
工序名称：精磨外圆
触发因素类型：精度类
具体触发因素：配套外圆精度要求
规则表达：若存在配套外圆精度要求，则出现精磨外圆工序
确认状态：已判稳
备注：若前道精车已满足要求，精磨外圆可不出现
```

### 2.2 示例：热处理类触发

```text
工序名称：去应力
触发因素类型：热处理类
具体触发因素：淬火后存在变形风险
规则表达：若零件经过淬火处理且长径比较大，则出现去应力工序
确认状态：已判稳
```

### 2.3 示例：待确认边界

```text
工序名称：研外圆
触发因素类型：精度类
具体触发因素：暂不确定
规则表达：（待确认）
确认状态：待确认边界
备注：已知该工序与配套外圆相关，但触发阈值未确认
```

## 3. 汇总输出结构

当所有工序对话完成后，系统应按以下结构输出本次规则汇总：

```text
# {零件族名称} 工艺路线规则提炼报告

## 已判稳规则

| 工序名称 | 触发因素类型 | 具体触发因素 | 规则表达 |
|---------|------------|------------|---------|
| 精磨外圆 | 精度类 | 配套外圆精度 | 若存在配套外圆精度要求，则出现精磨外圆 |
| 去应力   | 热处理类 | 淬火变形风险 | 若经过淬火且长径比大，则出现去应力 |

## 待确认边界

| 工序名称 | 已知信息 | 待确认项 |
|---------|---------|---------|
| 研外圆   | 与配套外圆相关 | 触发阈值未确认 |

## 本次对话统计

- 覆盖工序总数：{N}
- 已判稳工序：{N1}
- 待确认边界工序：{N2}
- 用户主动跳过工序：{N3}
```

## 4. 输出约束

- 每条规则的触发条件应尽量来自当前上传文档的实际证据，不应凭空推断
- 若某工序归因不稳，宁可标注待确认边界，不应强行生成确定性规则
- 规则表达优先使用“若…则…”格式，保持逻辑清晰
- 备注栏只在存在已知例外或边界说明时填写，不强制
