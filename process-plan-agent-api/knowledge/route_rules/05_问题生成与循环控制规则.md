# 问题生成与循环控制规则

## 1. 目标

本规则用于指导系统在第三步围绕“当前工序在什么条件下存在”生成问题，并驱动多工序的依次追问。

目标不是收集尽可能多的信息，而是用最少的问题帮助系统：

- 判断当前工序是否已经可判稳
- 判断当前工序主要受哪类因素驱动
- 找到当前工序最值得继续确认的边界
- 在多工序场景下形成完整闭环

## 1.1 第三步工序类别补充口径

当系统进入第三步规则分析时，先判断当前工序属于哪一类，再决定如何提问：

1. `独立工序`
   - 直接问：这道工序在什么条件下存在
2. `组合名工序`
   - 若名称中存在 `/`，先问统一名称
   - 若覆盖率不足 `100%`，再问：在什么条件下，这个工序应保留为统一名称 `X`

判断口诀：

- `名称不同，但系统已视为同一工序`：`组合名工序`
- `就是一个单独工序`：`独立工序`

## 2. 单工序问题生成流程

系统应按以下顺序生成问题：

1. 先判断当前是否仍存在关键未解释边界；若已可判稳或继续追问只会进入过细术语层，应优先停止
2. 先区分当前更像命名差异、记录差异、实际工艺差异，还是局部与整体 / 包含关系未澄清
3. 判断当前工序的优先归因方向
4. 优先围绕“什么条件下存在、什么条件下不存在”这一边界选题，而不是围绕工序名做泛泛提问
5. 生成一个归因问题，或直接进入对应中类问题
6. 从当前上传文档中抽取候选值
7. 为当前工序生成当前最值得问的一题
8. 若当前回答仍不足以解释边界，再进入下一问

补充说明：

- 下文各工序的“优先归因”只作为弱默认，不应压过当前任务中的正反样本边界、例外样本和已确认信息。
- 如果当前样本已经清楚说明边界，应优先沿样本边界发问，而不是机械套用工序名默认问题。
- 在进入“材料 / 结构 / 精度 / 热处理”等归因前，应先判断当前差异是否更像命名差异、记录口径差异，或局部与整体 / 上位工序包含关系未澄清。
- 不是所有工序都必须落到材料、尺度、结构、热处理这几类因素；若问题本质上只是叫法混用、记录合并或单列习惯不同，应优先沿这些方向收敛。

第三步补充：

- 若已判为 `组合名工序` 且正在问统一名称，不应提前跳到材料、结构、尺寸问题
- 若 `组合名工序` 已完成命名且覆盖率不足，则其后续问法按 `独立工序` 的覆盖不足追问模式执行

## 3. 工序优先归因规则

### 3.1 磨外圆 / 研外圆 / 精磨外圆

优先归因：

1. 精度或配套要求
2. 热处理状态
3. 结构因素

默认优先问题：

- 这道工序是否主要由配套外圆或更高外圆精度要求触发？

### 3.2 珩孔 / 研孔 / 精整孔类

优先归因：

1. 精度或配套要求
2. 孔类结构
3. 热处理状态

默认优先问题：

- 这道工序是否主要由配套内孔或更高孔精度要求触发？

### 3.3 钻孔 / 钻镗孔 / 镗孔 / 铰孔

优先归因：

1. 孔类结构
2. 精度要求
3. 小孔或深长孔要求

默认优先问题：

- 这道工序是否主要由孔类结构触发？

附加规则：

- 若工序名包含“镗”，优先提高精度因素权重
- 若工序名包含“铰”，优先提高孔精整因素权重

### 3.4 车槽 / 磨槽 / 槽加工类

优先归因：

1. 槽类结构
2. 精度要求
3. 可加工性约束

默认优先问题：

- 这道工序是否主要由槽类结构触发？

如果工序名包含“磨槽”，优先问题改为：

- 这道工序是否主要由高精度槽或密封槽要求触发？

### 3.5 中心孔加工

优先归因：

1. 中心孔结构
2. 基准或装夹需求

默认优先问题：

- 这道工序是否主要由中心孔定位或装夹需求触发？

### 3.6 线切割 / 电火花

优先归因：

1. 可加工性约束
2. 特殊结构
3. 尖边质量要求

默认优先问题：

- 这道工序是否主要因为常规加工方式难以直接完成该结构而增加？

### 3.7 去应力 / 时效

优先归因：

1. 热处理或应力状态
2. 变形风险
3. 长径比或细长件特征

默认优先问题：

- 这道工序是否主要由热处理变形风险或机加应力风险触发？

### 3.8 调质 / 正常化 / 淬火 / 渗氮 / 钝化

优先归因：

1. 材料相关
2. 热处理或表面处理要求

默认优先问题：

- 这道工序是否主要由材料和热处理要求触发？

## 4. 候选项与下一问规则

### 4.1 候选项抽取原则

候选值优先来自当前上传文档，其次才使用系统默认候选项。

抽取时优先关注：

- 材料牌号
- 热处理或表面处理名称
- 结构中类
- 精度或配套要求

### 4.2 下一问触发规则

如果用户已确认：

- 材料相关：下一问直接列出文档中出现的材料牌号
- 热处理相关：下一问直接列出文档中出现的热处理或表面处理方式
- 结构相关：先问结构中类，再问必要的二级类别
- 精度相关：优先问是配套外圆、配套内孔，还是更高表面质量或尺寸要求
- 可加工性相关：优先问是型孔或特殊开口、特殊尖边，还是常规方法难以完成

## 5. 单工序停止规则

当满足以下任一条件时，应停止继续追问当前工序：

1. 当前问题回答已能明显落成规则边界
2. 当前工序已可判稳
3. 当前问题继续追问只会进入过细特征库层
4. 用户连续两轮选择暂不确定
5. 当前继续追问已偏离该工序最关键的不确定边界

系统对每道工序默认只输出“当前最值得问的一题”，不应同时抛出多题并列混问。

## 6. 跨工序循环控制

系统应按以下顺序驱动整体循环：

1. 初始化工序待确认列表
2. 取出当前工序，进入单工序追问循环
3. 判断当前工序是否已判稳或转入待确认边界
4. 切换到下一道工序
5. 当待确认列表处理完成，或用户明确停止时，进入汇总输出

工序列表初始化时，应优先追问：

1. 按需工序
2. 知识库中无默认归因的工序
3. 文档证据不足的工序

主线工序通常可安排较后追问，或在证据充分时直接判稳。

## 7. 用户体验控制

### 7.1 及时告知进度

每次切换到新工序时，系统应简短告知当前进度。

### 7.2 允许用户跳过

若用户对某道工序暂时没有把握，应允许直接跳过，并将该工序标注为待确认边界继续推进。

### 7.3 允许用户提前终止

若用户提出停止，系统应立即输出当前已有的规则汇总，即使对话未全部完成。
